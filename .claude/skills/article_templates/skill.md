# Article Structure Templates (Skill)

This skill provides templates for designing technical article structures.
Used in Phase 2 (Outline Design Phase) of the article-writer agent.

## 1. Platform-Specific Template Structure

This skill supports three platforms.

### 1.1 Category Structure (CUSTOM - Generated by article-analyzer)

Categories are generated by the `article-analyzer` subagent based on your articles.
The following is an example structure.

**Example: Platform Categories**

| **{category_1}** | {description} | `templates/{platform}/{category_1}.md` |

> **Note**: Run `article-analyzer` to generate categories based on your own articles.

## 6. Notes for Outline Design

### 6.1 Platform-Specific Rules

**Qiita**
- Do not include links inside `:::note` blocks (place them outside)
- Write Solidity code as `js`
- Maximum 5 tags

**Zenn Article**
- Use `:::message` and `:::details` appropriately
- Maximum 5 topics
- Choose appropriate emoji

**Zenn Book**
- Chapter files use 6-digit random IDs (e.g., `a1b2c3.md`)
- Define table of contents structure in config.yaml
- Design each chapter to be readable independently
