# 言い回しパターン

- 技術や概念を定義するときは簡潔に述べる
  - 「[技術名]は、[定義]です」
  - 「[技術名]とは、[定義]を指します」

- 複数の特徴やポイントを紹介する前に宣言する
  - 「主要なポイントは以下の3つです」

- リストや項目を列挙する前は「次の」ではなく「以下の」を使う
  - 良: 「理由は以下の4つです」
  - 悪: 「理由は次の4つです」

- 操作や実装の手順を説明するときは順序を明示する
  - 「まず、[ステップ1]を行います」
  - 「次に、[ステップ2]を実施します」
  - 「最後に、[ステップ3]で完了します」

- 別のトピックに移るときは接続表現を使う
  - 「次に、[トピック]について説明します」
  - 「続いて、[トピック]を見ていきましょう」

- コードや図を提示する前に説明を入れる
  - 「具体的には、以下のようになります」

- 読者に特に注意してほしい点を強調する
  - 「重要なのは、[ポイント]です」
  - 「特に注目すべきは、[ポイント]です」

- 制約や例外を伝えるときは注意を促す
  - 「注意点として、[注意事項]があります」
  - 「ただし、[条件]の場合は[注意事項]です」

- 文体はです/ます調で統一する

- 技術用語の初出時は英語とカタカナを併記する
  - 「Account Abstraction（アカウント抽象化）」
  - 「Request for Quote（RFQ）」
