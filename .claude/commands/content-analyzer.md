description: 参考資料を6要素抽出フレームワークで分析
---

content-analyzerサブエージェントを起動して、以下のタスクを実行してください：

## タスク

$ARGUMENTS で指定された参考資料を6要素抽出フレームワークで分析し、JSON形式で返してください。

## 分析対象

$ARGUMENTS

形式例:
- URL: https://docs.example.com/guide
- パターン: ドキュメントベース/参考記事ベース/コードベース

## 6要素抽出フレームワーク

以下の6要素を抽出してください:

1. **テーマと主張**: 記事/ドキュメントの主要なテーマ
2. **問題/課題**: 解決しようとしている問題
3. **解決アプローチ**: 提案されている解決策
4. **実装/コード例**: 具体的な実装方法
5. **前提知識**: 必要な予備知識
6. **制約/トレードオフ**: 制限事項

## 期待される出力

```json
{
  "theme": {
    "title": "主要なテーマ",
    "summary": "テーマの要約"
  },
  "problem": {
    "description": "問題の説明",
    "context": "背景"
  },
  "solution": {
    "approach": "解決アプローチ",
    "key_concepts": ["概念1", "概念2", "概念3"]
  },
  "code_examples": [
    {
      "language": "JavaScript",
      "description": "説明",
      "code": "コード例"
    }
  ],
  "prerequisites": ["前提知識1", "前提知識2"],
  "tradeoffs": ["トレードオフ1", "トレードオフ2"]
}
```

## 重要な制約

- 結果はJSON形式のみで返す（mainのコンテキストを圧迫しない）
- 不明な要素はnullとして記録（推測しない）
- WebFetchは必要最小限の情報のみ取得
